Tokens are base units and settings that underpin the whole design system. Everything is based off of these.

  • Base size and the sizing scale
  • Base colour, secondary colour and neutral colour and the tints thereof.
  • Shadows, focus styles.
  • Fonts, line-height, max-line-length.

Icons

This pattern describes how an icon should be marked-up. It doesn't have any classes. There are different usage and fallback scenarios to consider.

Emphasis icon

An in-line right-pointing chevron icon: (note this will be missing if SVG is not supported (NoSVG)).

Show code: 
<svg focusable="false" aria-hidden="true" width="1.25em" height="1.25em"><use xlink:href="#icon-chevron-right"></use></svg>

In this scenario the icon adds visual emphasis to a link (CTA) - it does not add any information so can be accessibly-hidden with aria-hidden="true":

Call to Action

Show code: 
<a href="#" class="c-cta">
    Call to Action<svg focusable="false" aria-hidden="true" width="1.25em" height="1.25em" display="none"><use xlink:href="#icon-chevron-right"></use></svg>
</a>

Note in the example above we've added display="none" to the SVG. Most often these 'Emphasis Icons' should be hidden if CSS is unavailable.

For example, with Info-Overlay a details/summary element is styled such that an icon acts as the open/close button, but if styles aren't loaded the icon looks out-of-place. Similarly the CTA chevron above positions itself badly in NoCSS scenario and is better not shown at all.

Note that buttons not added via JS are better off having the hidden attribute instead.

Test things on a case-by-case basis, if the icons should appear in NoCSS scenarios, then remove the display="none".

Solo icon

An in-line right-pointing search icon: (note in a NoSVG scenario the word 'Search' will appear instead).

In this scenario the icon is the only thing visible so an accessible label needs to be present, and text needs to be visible for NoSVG scenarios:

Show code: 
<button aria-labelledby="search-button-label">
    <svg focusable="false" aria-hidden="true" width="1.25em" height="1.25em"><use xlink:href="#icon-search"></use><text visibility="hidden" id="search-button-label">Search</text></svg>
</button>

Another example:

Show code: 
<button id="env-msg-close" aria-labelledby="env-msg-close-button-label" onclick="this.parentNode.parentNode.parentNode.style.display='none'">
    <svg focusable="false" aria-hidden="true" width="1.25em" height="1.25em"><use xlink:href="#icon-cross"></use><text visibility="hidden" id="env-msg-close-button-label">Close this notice</text></svg>
</button>
