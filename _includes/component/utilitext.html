This pattern deals with the appearance and behaviour of utilitarian text. Use the layout pattern "text-group" to handle any layout concerns.

If a set of links should be considered a list, use list markup, unless it should fall back to an inline-list, in which case use this:

<p role="list">
    <span role="listitem"><a ... />&nbsp;|</span>
    <span role="listitem"><a ... /></span>
    ...
</p>

A this example re-done using layout classes can be found below.

Notice how the nesting requirements for the layout pattern (p.l-layout > spans.l-layout__inner) would cause a problem with normal list markup (you'd have to include an outer wrapper) are not a problem using the ARIA syntax.

Using layout classes

An example adding layout classes:

Home | Contact Us | Privacy and Cookies | Accessibility | Data Protection | Top 

<p role="list" class="l-layout  l-row  l-row--start  l-gutter--s  l-flush-edge-gutter">
    <span class="l-layout__inner">
        <span role="listitem" class="l-box">
            <a ...><span>Home</span></a><span class="l-box__separator">&nbsp;|</span>
        </span>
        <span role="listitem" class="l-box">
            <a ...><span>Contact Us</span></a><span class="l-box__separator">&nbsp;|</span>
        </span>
        ...
        <span role="listitem" class="l-box">
            <a ...><span>Top</span></a>
        </span>
    </span>
</p>

Notice how the nesting requirements for the layout pattern (p.l-layout > spans.l-layout__inner) would cause a problem with normal list markup (you'd have to include an outer wrapper) are not a problem using the ARIA syntax.
